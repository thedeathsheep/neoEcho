# 项目代号 Echo 技术叙事

## 核心判断

项目代号 Echo 的技术差异化，不在于模型更强，而在于我们把写作理解为一个持续运行的状态系统，而不是一次次调用模型的问答过程。

今天的大多数 AI 写作产品，底层都是：

`输入 -> 检索 -> 拼提示词 -> 调模型 -> 返回结果`

而 Echo 的目标技术形态是：

`文档事件 -> 记忆更新 -> 后台代理 -> 候选界面 -> 用户采纳 -> 记忆反馈`

这不是“调用 API 的应用层包装”，而是面向写作过程的运行时。

## 我们为什么不是普通 RAG 产品

### 1. 检索不是目的，记忆才是目的

普通 RAG 关注“找到相关片段”，Echo 关注“形成作者记忆”。

我们未来要维护的不只是文档切块和 embedding，还包括：

- 作品记忆：人物、场景、主题、意象、时间线
- 偏好记忆：作者偏爱的句法、节奏、叙事距离
- 修改记忆：作者经常返工的位置、常见问题和修订模式
- 采纳记忆：哪些回声被使用、哪些建议被忽略

因此，知识库的终点不是检索命中率，而是长期理解作者。

### 2. 智能不是回答问题，而是参与写作状态

普通 AI 产品把智能放在聊天框里；Echo 让智能进入写作环境本身。

- 织带负责提供环境式回响
- 详情面板负责完整展开
- 写作台负责诊断、修订和积累
- 背景任务负责在不打断用户的前提下更新候选与记忆

我们的重点不是“对话体验”，而是“写作过程中的低打断参与感”。

### 3. 架构不是 stateless assistant，而是 stateful runtime

用户每次写作都不是一次新会话，而是在已有作品状态上继续推进。

因此系统需要天然支持：

- 文档事件驱动
- 长期状态保存
- 多源记忆融合
- 用户采纳后的反馈回流
- 后台低可见代理的持续运行

这让 Echo 更接近一个 authoring runtime，而不是一个聊天式 copilot。

## 我们的目标技术形态

### 1. Local-first

本地优先不是部署方式，而是产品原则。

- 文稿与写作过程尽量留在本地
- 知识库与作者记忆优先在本地组织
- 外部模型调用是能力层，而不是数据主存层

这既是隐私策略，也是体验策略，因为写作是高私密、高连续性的活动。

### 2. Event-driven

系统的核心驱动不应是“用户按下生成按钮”，而应是文档事件。

关键事件包括：

- 文本更新
- 停顿
- 章节切换
- 回声点击
- 建议采纳
- 修订记录
- 素材导入

每个事件都应该推动记忆状态和候选状态发生变化。

### 3. Memory-native

记忆不是某个附属模块，而是整个系统的中间层。

无论是织带、结构诊断、人物一致性还是修订建议，都应该优先从记忆层读写，而不是各自临时拼 prompt。

未来最重要的技术资产不是提示词集合，而是作者记忆模型。

### 4. Ambient agents

我们不做一个显眼的大 Agent，而做多个低可见后台代理。

建议中的代理切分：

- 织带代理：维护回声池与候选流
- 结构代理：感知段落与章节层面的结构问题
- 人物代理：维护角色特征、一致性与关系线索
- 修订代理：维护 revision queue 与问题聚类
- 素材代理：将选中片段、回声、摘录沉淀为可回用材料

这些代理的价值不在“替用户做决定”，而在“持续保持写作环境有感知能力”。

## 对外可讲的四层架构

### 文档层

承载写作对象本身。

- 编辑器
- 章节与场景
- 快照
- 修订清单
- 素材片段

### 记忆层

承载产品真正的长期价值。

- 知识库
- 作品记忆
- 偏好记忆
- 修改记忆
- 采纳记忆

### 代理层

承载动态分析与候选生成。

- 灵感代理
- 结构代理
- 人物代理
- 修订代理
- 素材代理

### 界面层

承载用户可感知的输出形式。

- 织带
- 详情面板
- 写作台
- 素材池
- 修订面板

## 技术壁垒该怎么讲

### 可以讲的壁垒

- 本地优先的作者记忆系统
- 面向写作过程的事件驱动运行时
- 环境式交互，而非聊天式交互
- 作者采纳与修订行为的持续反馈闭环

### 不该当核心壁垒讲的东西

- 兼容多少家模型
- RAG 调得多复杂
- 提示词写得多长
- UI 里堆了多少 AI 功能点

这些都容易被复制，也不足以形成真正的产品身份。

## 对内的技术原则

- 优先构建状态系统，不优先堆提示词
- 优先构建记忆层，不优先堆模型接口
- 优先构建低打断交互，不优先做显眼 Agent
- 优先让产品越来越懂作者，不优先让产品看起来越来越能生成

## 一句话技术口径

项目代号 Echo 不是把模型接进编辑器，而是为写作构建一个本地优先、事件驱动、具备长期记忆的 authoring runtime。
